## 日本节点一共两个，不能混着看

这是 Evoxt 日本 VPS 最容易被人忽略的地方——**东京和大阪不是同一类机房**。

### 东京机房：普通线路，原生 IP 是亮点

东京机房接入 BBIX 和 JPIX，主要 transit 走 Softbank。硬件采用 AMD EPYC Genoa 系列处理器，单核频率约 3.7 GHz，磁盘 I/O 实测接近 2.6 GB/s，性能非常扎实。

网络方面，对国内用户来说有明显的"绕路"问题：电信和联通去回程基本都经过美国中转，平均延迟在 300ms 以上，移动去程走香港直连稍好一些，但回程也绕。所以如果你的需求是低延迟访问中国大陆用户，东京机房并不合适。

但东京有一个硬核优势——**日本原生 IP 解锁能力极强**。实测可以解锁日本 Netflix、Disney+、Amazon Prime Video、TikTok（日区）、部分 ChatGPT 和 Gemini 服务，这在非住宅 IP 的机房里属于少见的水准。如果你的需求是部署中转节点、解锁日本流媒体服务、或者面向日本本土用户提供服务，东京机房是个划算的选择。

### 大阪机房（Premium Network）：软银直连，流媒体全解锁

大阪机房被 Evoxt 标注为"优质网络（Premium Network）"，走的是日本软银（Softbank / BBTEC）线路。从国内三网来看：

- **电信、移动**：可以直连或经由香港转到日本软银，路由比东京机房友好得多，延迟约 60～90ms 区间
- **联通**：丢包率相对较小，但绕道后延迟仍偏高（200ms+）

截至 2026 年 6 月，大阪软银机房近半年稳定性有所波动，电信和移动均出现一定丢包现象，建议购买前先用测试 IP 验证当前网络质量再决定。

流媒体解锁是大阪机房的另一个看点：IPv4 实测可全原生解锁 Netflix、Disney+、Amazon Prime、ChatGPT、Gemini，属于"全家桶"水平，在机房 IP 里很难得。

相比东京，大阪机房在相同价格下，月流量配额减半（属于优质网络区域的统一规则），但线路质量对国内用户更友好。
